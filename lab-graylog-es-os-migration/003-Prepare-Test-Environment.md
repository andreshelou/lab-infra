# Step 1 - Create a Test Data Generator

## Objective

Create a continuous Syslog message generator that will be used during the complete migration validation.

The generated messages must be easy to identify from Graylog and must contain enough information to validate the migration.

---

## Requirements

The generator must:

- Send Syslog messages over UDP.
- Generate messages continuously.
- Include a sequential number.
- Include the hostname.
- Include the current timestamp.
- Use a unique prefix to simplify searches.

---

## Message Format

Example

```
LAB-MIGRATION sequence=1 hostname=slk timestamp=2026-07-22T18:30:01
```

Every generated message should contain the same structure.

---

## Validation

The messages must:

- Arrive at Graylog.
- Be searchable.
- Preserve their timestamp.
- Preserve their hostname.
- Be easy to identify using the prefix:

```
LAB-MIGRATION
```

---

## Expected Result

Graylog continuously receives test messages generated by the laboratory.

---

## Evidence

A successful search for:

```
LAB-MIGRATION
```

must return the generated messages.
