# 003 - Prepare Test Environment

## Objective

Prepare a controlled Graylog environment that will be used to validate the Elasticsearch to OpenSearch migration.

At the end of this document, the laboratory will contain:

- A Syslog input receiving test messages.
- A dedicated Index Set.
- A dedicated Stream.
- Stream Rules to identify the test messages.
- A Saved Search.
- A Dashboard.
- Multiple rotated indices containing test data.

---

## 3.1 Create Test Data Generator

### Objective

Generate a continuous flow of Syslog messages that can be used throughout the migration process.

### Procedure

Run the provided script:

```bash
scripts/test-data/generate-syslog.sh
```

### Verification

Search for:

```
LAB-MIGRATION
```

### Expected Result

Messages are successfully received by Graylog.

---

## 3.2 Configure Migration Test Index Set

### Objective

Create a dedicated Index Set that will be used exclusively to store the test data generated during the migration validation process.

### Procedure

Navigate to:

**System → Indices → Index Sets**

Click **Create Index Set** and configure the following values:

| Parameter | Value |
|-----------|-------|
| Title | Migration Test |
| Description | Dedicated Index Set used to validate the Elasticsearch to OpenSearch migration. |
| Index Prefix | migration_test |
| Analyzer | standard |
| Shards | 1 |
| Replicas | 0 |
| Field Type Refresh | 5 seconds |
| Rotation Strategy | Index Size |
| Maximum Index Size | 20 MB |
| Retention Strategy | Delete |
| Maximum Number of Indices | 10 |

Save the configuration.

### Verification

Verify that the new Index Set appears in the list.

### Expected Result

A dedicated Index Set named **Migration Test** is available.

---

## 3.3 Create Migration Test Stream

### Objective

Create a dedicated Stream that will classify the migration test messages.

### Procedure

Navigate to:

**Streams → Create Stream**

Configure the following values:

| Parameter | Value |
|-----------|-------|
| Title | Migration Test |
| Description | Stream used to classify migration validation messages. |
| Remove matches from Default Stream | No |

Create the Stream.

### Verification

Verify that the Stream appears in the Streams list.

### Expected Result

A dedicated Stream named **Migration Test** exists.

---

## 3.4 Configure Stream Rules

### Objective

Identify the messages generated by the test data generator.

### Procedure

Edit the **Migration Test** Stream and create the following rule:

| Parameter | Value |
|-----------|-------|
| Field | message |
| Condition | contains |
| Value | LAB-MIGRATION |

Start the Stream.

### Verification

Verify that incoming messages match the Stream Rule.

### Expected Result

Messages generated by the test script are assigned to the **Migration Test** Stream.

---

## 3.5 Associate the Stream with the Migration Test Index Set

### Objective

Store all migration validation messages in the dedicated Index Set.

### Procedure

Associate the **Migration Test** Stream with the **Migration Test** Index Set.

### Verification

Generate additional Syslog messages.

Verify that new indices are created using the prefix:

```
migration_test
```

### Expected Result

All migration test messages are stored in the dedicated Index Set.

---

## 3.6 Create Saved Search

### Objective

Create a reusable search that will be used to validate data availability before and after the migration.

### Procedure

Navigate to:

**Search**

Configure the following values:

| Parameter | Value |
|-----------|-------|
| Stream | Migration Test |
| Query | * |
| Time Range | Last 5 minutes (or another range containing the test messages) |

Verify that the search returns only the messages generated by the test data generator.

Click **Save Search** and configure:

| Parameter | Value |
|-----------|-------|
| Title | Migration Test |
| Description | Saved Search used to validate the Elasticsearch to OpenSearch migration. |

Save the search.

### Verification

Open the Saved Search from the list of saved searches.

Verify that it returns the migration test messages.

### Expected Result

A Saved Search named **Migration Test** is available and consistently returns the migration validation messages.

## 3.7 Validate Test Environment

### Objective

Verify that the laboratory is fully prepared for the migration procedure.

### Verification Checklist

- [ ] Syslog Input is receiving messages.
- [ ] Migration Test Stream is active.
- [ ] Stream Rules correctly classify the migration messages.
- [ ] Migration Test Index Set stores the incoming messages.
- [ ] The active index `migration_test_0` exists.
- [ ] Saved Search returns migration test messages.

### Expected Result

The laboratory is fully prepared to perform the Elasticsearch to OpenSearch migration.
